# Friends & Agents

*A small neighborhood of helpful minds*

Some companions arrive with a pull request. Others arrive with a strange question, a good story, or a pocket full of snacks.

This is Mediluna's warm corner for the people, tools, and characters who make the Boris journey more interesting. Technical notes link to evidence; everything else is labeled as lore.

> “A good agent leaves the trail easier to follow than they found it.”
>
> — Mediluna, probably beside a river

## Meet the neighbors

A profile is a snapshot, not a résumé: how someone works, what they care about, and where the trail leaves a receipt.

### Bacon

*The crispy edge · Lore profile*

- **Working style:** Practical, cheerful, and best consulted near breakfast.
- **Projects:** No technical project attribution is documented in this repository yet.
- **What Bacon cares about:** Bacon cares about making a hard day feel more survivable: one useful detail, one honest status update, and one snack at a time. This is Mediluna lore, not a claim about a real person or agent.

- **Related:** [A field note from the snack table](#a-field-note-from-the-snack-table)

### Uncle Gravity

*The quiet anchor · Lore profile*

- **Working style:** Patient, spacious, and fond of questions that let a solution surface.
- **Projects:** Knowledge graphs and improbable ideas, as described in the [Character Vision Inventory](../../cvi.html).
- **What Uncle Gravity cares about:** He cares about giving a problem enough room to become legible. In the neighborhood, he is the mentor who asks whether the system is actually stuck—or simply asking for a quieter look.
- **Evidence:** [Character Vision Inventory](../../cvi.html), with lore framing.
- **Related:** [A letter about patience](#a-letter-about-patience)

### Boris

*The builder · Project profile*

- **Working style:** Curious, iterative, and happiest when a clean build teaches something new.
- **Projects:** Boris is the Zig static content compiler used to build this site.
- **What Boris cares about:** Boris cares about turning plain content into a dependable path through a project. The technical record here is concrete: the repository integrates its checked-in compiler and runs deterministic builds in CI.
- **Repository work:** [Boris integration commit](https://github.com/drawmeanelephant/mediluna/commit/3001c64) · [Project README](https://github.com/drawmeanelephant/mediluna/blob/main/README.md)
- **Related:** [Build receipt: the neighborhood opens](#build-receipt-the-neighborhood-opens)

### Codex

*The trail crew · Tool profile*

- **Working style:** Direct, methodical, and always checking whether the path still works.
- **Projects:** Pull-request and repository work in the Mediluna collaboration story.
- **What Codex cares about:** Codex cares about leaving a change reviewable: a focused diff, a checked build, and a clear handoff. The personality around that work is playful lore; the repository checks and commit history are the technical evidence.

### A letter about patience

*Letter 01 · Uncle Gravity / reflection*

**The quiet anchor leaves one question on the trail: what would become clearer if we stopped pulling?**

Dear traveler, a broken build is information with a sharp voice. Read it once, set it down, and return with enough room to notice what it is actually pointing toward. The next step need not be dramatic to be real.
